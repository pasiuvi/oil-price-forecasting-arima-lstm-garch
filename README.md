# Brent Crude Oil — Price Forecasting & Volatility Analytics

Comparing traditional statistical, machine learning, and deep learning
approaches to forecasting Brent Crude Oil prices, paired with GARCH-based
volatility modelling — built as an end-to-end pipeline from raw data to a
deployed, interactive dashboard.

**Live dashboard:** https://oil-price-forecasting-arima-lstm-garch.streamlit.app/
**Repository:** https://github.com/pasiuvi/oil-price-forecasting-arima-lstm-garch

---

## Why this project

Brent Crude is one of the two global oil price benchmarks and feeds directly
into freight and fuel cost planning across logistics and supply chain
operations. Unlike a generic stock index, this series combines a clear
long-term trend, irregular event-driven shocks (the 2020 price collapse, the
2022 spike), and strong volatility clustering — a genuinely useful test case
for comparing forecasting approaches rather than a purely academic exercise.

## Key results

| Model | MAE | RMSE | MAPE |
|---|---|---|---|
| **ARIMA** | 0.454 | **0.708** | 0.73% |
| Naive Baseline | 0.453 | 0.713 | 0.73% |
| Random Forest | 0.536 | 0.725 | 0.86% |
| LSTM | 0.766 | 0.916 | 1.23% |

At a 1-day-ahead horizon, no model meaningfully beats a naive "tomorrow
equals today" baseline — a finding consistent with the Efficient Market
Hypothesis, and a more interesting result than it might first appear (see
the full write-up for why).

**Volatility modelling:** a GJR-GARCH(1,1) model was selected over standard
GARCH(1,1) (AIC 15751.9 vs 15787.9), confirming an asymmetric "leverage"
effect — negative price shocks increase future volatility more than
equally-sized positive shocks.

## Project structure

```
src/
  01_download_data.py          Pulls Brent Crude data from Yahoo Finance
  02_eda_preprocessing.py      EDA, stationarity testing, feature engineering
  03_model_development.py      ARIMA, LSTM, Random Forest — trained & compared
  04_garch_volatility.py       GARCH(1,1) / GJR-GARCH volatility modelling
  05_final_comparison_summary.py   Pulls results together for reporting
dashboard/
  app.py                       Interactive Streamlit dashboard
data/                          Raw and processed data (generated by scripts)
outputs/                       Plots and result CSVs (generated by scripts)
requirements.txt               Full pipeline dependencies
dashboard/requirements.txt     Lightweight dependencies for cloud deployment
INSTRUCTIONS.md                Full setup and run guide
```

## Models implemented

| Category | Model |
|---|---|
| Statistical baseline | Naive persistence |
| Statistical | ARIMA (rolling one-step-ahead) |
| Deep learning | LSTM |
| Machine learning ensemble | Random Forest |
| Volatility | GARCH(1,1), GJR-GARCH(1,1) |

## Quick start

See [INSTRUCTIONS.md](INSTRUCTIONS.md) for the complete step-by-step guide,
including Python environment setup, running the full pipeline, and deploying
the dashboard.

```bash
pip install -r requirements.txt
cd src
python 01_download_data.py
python 02_eda_preprocessing.py
python 03_model_development.py
python 04_garch_volatility.py
python 05_final_comparison_summary.py
cd ../dashboard
streamlit run app.py
```
